# Prompt Library

Natural One Society Prompt Library

Status: Active

Version: 1.0

## Purpose

The Prompt Library preserves approved prompts, workflows, templates, and reusable operational procedures.

The Prompt Library exists to reduce repetitive work, improve consistency, and accelerate future sessions.

The Prompt Library is a preservation and workflow resource.

The Prompt Library does not establish canon.

Authority remains with approved Canon documents and Operating Procedures.

The Prompt Library helps answer:

* What prompts are commonly used?
* What workflows are repeatable?
* How should common tasks be performed?
* How can future sessions start more efficiently?
* How can successful processes be reused?

## Prompt Categories

Natural One Society prompts may be organized into the following categories:

### Session Prompts

Prompts used to initialize, recover, or continue project work.

### Workflow Prompts

Prompts used to perform repeatable project tasks.

### Character Development Prompts

Prompts used during character creation and refinement.

### Canon Management Prompts

Prompts used to review, validate, and maintain canon.

### Repository Management Prompts

Prompts used to manage repository structure, documentation, and preservation.

## Prompt Library Usage

Prompts should be added when they demonstrate consistent value and are likely to be reused.

Experimental prompts may be tested before inclusion.

Approved prompts should be documented clearly enough that future sessions can execute the workflow without requiring historical chat access.

The Prompt Library preserves successful workflows.

The Prompt Library does not replace Operating Procedures.

## Navigation

Use the Prompt Library when researching:

* Startup procedures
* Session recovery
* Character development workflows
* Canon review workflows
* Repository workflows
* Reusable operational processes

The Prompt Library should be consulted when a task has been performed successfully before and may benefit from standardization.

## Core Philosophy

Reusable knowledge should be preserved.

Successful workflows should be documented.

Repeated tasks should be simplified.

The goal is not to automate creativity.

The goal is to eliminate unnecessary repetition.

The Prompt Library preserves operational knowledge.
