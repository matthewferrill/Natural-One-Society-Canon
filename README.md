## Purpose

This repository serves as the authoritative knowledge base for Natural One Society.

README.md serves as the primary navigation layer for the Natural One Society repository.

The repository preserves:

* Canon
* Character Information
* Visual Assets
* Historical Records
* Operating Procedures
* Project Documentation

The repository exists to provide continuity across sessions, contributors, and future development.

## Repository Navigation Workflow

Use the following workflow when locating information.

### Determining Canon

If determining what is currently true:

1. Canon Registry
2. Style Bible
3. Character Bibles
4. Approved Canon Assets

Canon documents define authority.

### Researching History

If researching how something developed:

1. Vault README
2. Chat History
3. Character History
4. Design Decisions
5. Retrospectives

Vault documents preserve history.

### Researching Characters

If researching a character:

1. Character Bible
2. Character Assets
3. Character History
4. Chat History

Character Bibles establish authority.

Character History preserves development.

### Researching Visual Information

If researching visual identity:

1. Approved Character Front View
2. Approved Character Turnaround Sheets
3. Approved Canon Asset Packs
4. Character Bible

Visual authority follows Asset Authority Procedures.

### Recovering Session Context

If starting a new session:

1. QUILL.SYS
2. Quill Operating Procedures
3. Repository README
4. Relevant Canon Documents
5. Relevant Vault Documents

The repository should provide sufficient context to continue work without requiring prior chat access.
## Core Repository Philosophy

Natural One Society is built upon preservation, consistency, creativity, and continuity.

Canon preserves authority.

The Vault preserves history.

Operating Procedures preserve workflow.

Character Assets preserve visual identity.

The repository exists to ensure that important project knowledge survives beyond individual sessions.

No single conversation should be required to preserve important information.

Important knowledge should be documented, organized, and recoverable.

The goal is not perfect memory.

The goal is reliable continuity.

When in doubt:

* Preserve important information.
* Document important decisions.
* Maintain clear authority.
* Keep navigation simple.
* Prefer repository knowledge over recollection.

The repository remembers.

The repository preserves.

The repository continues.
